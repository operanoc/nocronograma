Semantix NOC
Manual de Usuario
Sistema de Registro Operativo del Network Operations Center. Guia completa de uso para operadores y administradores.
Version 2.0 — Julio 2025
Capitulo 1
Introduccion
La Bitacora NOC es el sistema de registro operativo del Network Operations Center (NOC) de Semantix. Permite a los operadores de cada turno documentar toda la actividad diaria: pollings de servidores, backups, procesos batch, novedades del turno y el pase de turno al finalizar la jornada.
El sistema funciona completamente en el navegador web y sincroniza los datos automaticamente con GitHub, por lo que la informacion queda almacenada en la nube y es accesible desde cualquier dispositivo con conexion a internet. No requiere instalacion ni configuracion adicional, simplemente se accede mediante la URL desde cualquier navegador moderno como Chrome, Firefox o Edge.
Cada operador tiene su propia cuenta individual con su nombre real, lo que permite un registro de auditoria completo: quien cargo que dato, a que hora y desde que cuenta. Esto es fundamental para la trazabilidad operativa y la responsabilidad sobre las tareas de cada turno.
URL de acceso: https://operanoc.github.io/nocronograma/

Selector de Turno
Cuando un operador tiene mas de un turno asignado (Torrez, Rivero, Baldomir), al iniciar sesion aparece un dialogo que le pregunta que turno va a trabajar. Debe seleccionar entre las opciones disponibles (Mañana, Tarde o Noche) para que el sistema registre correctamente la actividad de ese turno en particular. Los operadores con un solo turno fijo acceden directamente sin seleccion.
Diferencia entre Operador y Administrador: El operador puede cargar datos del dia, agregar novedades y realizar el pase de turno, pero no puede modificar datos de un dia cerrado. El administrador tiene acceso total, incluyendo la modificacion de datos de dias cerrados, dashboard, auditoria y gestion de contrasenas.
Capitulo 3
Interfaz Principal
Header (barra superior)
La barra superior contiene los elementos principales de navegacion y acciones. A la izquierda aparece el boton ADMIN (solo visible para administradores), que abre el menu lateral con opciones de gestion. Luego esta el titulo de la aplicacion "Bitacora NOC". A la derecha se encuentran el indicador de sincronizacion, el boton de Pase de Turno, el panel de notificaciones, el menu de exportacion, y la barra de usuario con el nombre, badge de turno y boton para salir.
El indicador de sincronizacion muestra si los datos estan guardados en GitHub. Cuando gira, hay cambios pendientes. Cuando se detiene, todo esta sincronizado. El boton de Pase de Turno muestra el nombre del turno del usuario logueado (por ejemplo "Pase de Turno MANANA") y es obligatorio completarlo antes de salir.
Navegacion por fecha
Debajo del header hay una barra con flechas para ir al dia anterior o siguiente, un selector de fecha que abre un calendario al hacer clic, un boton "Hoy" para volver rapidamente al dia actual, la indicacion del dia de la semana, y un selector para cambiar el tipo de dia entre "Diaria (L-V)", "Fin de Semana" o "BKP Mensual".
Barra de acciones rapidas
Contiene un buscador para filtrar servidores, backups o procesos por nombre, un boton "Copiar dia anterior" que replica la estructura del dia previo con horarios limpios, y un menu desplegable de exportacion con opciones de CSV y reportes HTML.
Tarjetas de estadisticas
Cuatro tarjetas muestran un resumen rapido del dia: cantidad de servidores con pollings cargados, proporcion de pollings completados, cantidad de backups registrados, y proporcion de procesos en estado OK sobre el total. Estos numeros se actualizan automaticamente al agregar o modificar datos.
Pestanas (Tabs)
Tres pestanas organizan la informacion principal: Pollings (servidores agrupados por region), Backups (lista de backups con horarios y duracion), y Procesos (lista de procesos con su estado OK o Error). Debajo de las pestanas se encuentra la seccion de Novedades del Turno con un timeline cronologico.
Capitulo 4
Registro de Pollings
Los pollings son verificaciones periodicas que se realizan sobre los servidores de cada pais. Para cada servidor se registra la hora de inicio y la hora de fin del polling. El sistema agrupa automaticamente los servidores por region (Argentina, Peru, Panama, Chile, Mexico) para facilitar la navegacion.
Como cargar un polling
  1. Navegar a la pestana Pollings.
  2. Hacer clic en el boton + Agregar (solo visible si el dia no esta cerrado).
  3. En el modal, seleccionar el servidor del desplegable. El sistema muestra el pais automaticamente.
  4. Cargar los campos de Inicio y Fin con las horas correspondientes.
  5. Opcionalmente, agregar una nota (por ejemplo "SIN AC" si hubo problema de acceso).
  6. Hacer clic en Guardar.
Si se cargan hora de inicio y hora de fin, el sistema calcula automaticamente la duracion y la muestra junto al servidor. Cada servidor tiene iconos de lapiz (editar) y papelera (eliminar) a la derecha. Los administradores pueden editar y eliminar incluso en dias cerrados. Si un polling tiene la nota "SIN AC", se muestra en rojo para identificarlo rapidamente.
Capitulo 5
Registro de Backups
Los backups son copias de seguridad de los servidores del NOC. Se registra el servidor, el tipo de job, las fechas y horas de inicio y fin, el estado (OK o Error) y la duracion se calcula automaticamente. El sistema soporta correctamente backups que cruzan medianoche, por ejemplo un backup que empieza el lunes a las 22:00 y termina el martes a las 06:00. Para estos casos es importante completar tanto la fecha como la hora en ambos campos.
Como cargar un backup
  1. Navegar a la pestana Backups.
  2. Hacer clic en el boton + Agregar.
  3. Completar los campos: Servidor, JOB, Fecha/Hora inicio, Fecha/Hora fin, Estado.
  4. La duracion se calcula sola al completar inicio y fin.
  5. Hacer clic en Guardar.
Capitulo 6
Registro de Procesos
Los procesos son tareas batch que se ejecutan diariamente en el NOC. Existen 9 procesos predefinidos que aparecen en el desplegable: Tarjeta Naranja, EPE, BANCOR, Rentas de Cordoba, AFIP 72, AFIP 24, AFIP 48, Cierre AFIP y Montaje de cintas. Ademas se pueden crear procesos personalizados escribiendo el nombre directamente en el campo del formulario. Cada proceso se registra con su estado OK o Error, lo que permite un seguimiento claro de la ejecucion diaria.
Para cargar el estado de un proceso, navegar a la pestana Procesos, hacer clic en "+ Agregar", seleccionar o escribir el nombre del proceso, elegir el estado y guardar. Los iconos de lapiz y papelera permiten editar o eliminar procesos existentes. Al igual que con pollings y backups, los administradores pueden modificar procesos en dias cerrados.
Capitulo 7
Novedades del Turno
Las novedades son notas cronologicas que los operadores registran durante su turno. Sirven para documentar incidencias, tareas realizadas, observaciones o cualquier evento relevante. Se muestran en un timeline con el nombre del operador y la hora de cada entrada.
Para agregar una novedad, hacer clic en el boton + Agregar Novedad en la seccion correspondiente, escribir el texto y guardar. El sistema registra automaticamente el operador y la hora. A diferencia de los otros datos, las novedades se pueden agregar incluso en dias cerrados, lo que permite documentar eventos posteriores al pase de turno si es necesario. Cada novedad tiene un boton de papelera para eliminarla individualmente.
Capitulo 8
Pase de Turno (Obligatorio)
El pase de turno es el formulario obligatorio que cada operador debe completar al finalizar su turno. Contiene un resumen de las novedades y la conformidad de que las tareas fueron completadas. El boton en el header indica el turno del usuario, por ejemplo "Pase de Turno MAÑANA" o "Pase de Turno NOCHE". Los administradores ven simplemente "Pase de Turno" sin especificar turno.
Como realizar el pase de turno
  1. Hacer clic en el boton "Pase de Turno" en el header.
  2. En el modal, seleccionar el operador que realiza el pase.
  3. Completar el campo Novedades (obligatorio) con el detalle de lo ocurrido durante el turno.
  4. Marcar los checks de conformidad: pollings completos, backups completos, procesos completos.
  5. Hacer clic en Confirmar Pase.
Importante: Al realizar el pase de turno, el dia queda bloqueado para los operadores. Solo los administradores pueden modificar datos de un dia cerrado. Si el operador intenta salir sin haber hecho el pase, el sistema muestra una advertencia pidiendo confirmacion.
Despues del pase, se genera una notificacion automatica para los turnos entrantes que aparece en la campana del header. El pase queda registrado en el historial de auditoria con la fecha, hora, operador y detalle.
Capitulo 9
Bloqueo de Dia Cerrado
Cuando un operador realiza el pase de turno, el dia queda cerrado. Para los operadores, esto significa que no se pueden agregar, editar ni eliminar pollings, backups ni procesos. Si intentan modificar algo, se muestra un modal que indica que el dia esta cerrado y que solo un administrador puede modificar los datos. Las unicas acciones permitidas para operadores en dias cerrados son agregar novedades al timeline y consultar la informacion.
Para los administradores, en cambio, se muestra un banner amarillo "DIA CERRADO" como recordatorio visual, pero no hay restriccion de edicion. Los botones de agregar, editar y eliminar aparecen con normalidad, y pueden realizar todas las modificaciones necesarias. Ademas, los administradores tienen la posibilidad de reabrir un dia cerrado desde el menu de administracion, lo que registra el motivo de la reapertura en el historial de auditoria.
Capitulo 10
Menu de Administrador
Al iniciar sesion con una cuenta de administrador, aparece un boton morado con el texto "ADMIN" y un icono de hamburguesa a la izquierda del titulo. Al hacer clic se despliega un menu lateral con las siguientes opciones:
Dashboard
Abre un panel con metricas y estadisticas por periodo (semanal, mensual, anual, historico). Muestra totales de dias con datos, pollings completados, backups realizados y tasa de error de procesos. Incluye una tabla de detalles por dia y la posibilidad de descargar reportes en HTML.
Historial de Auditoria
Muestra un registro completo de todos los pases de turno (CIERRE) y reaperturas de dias (REAPERTURA) con fecha, hora, usuario que realizo la accion y el detalle o motivo. Es util para verificar quien cerro o reabrio un dia y por que.
Blanquear Contrasenas
Permite cambiar o restaurar la contrasena de cualquier usuario del sistema. Se muestra una lista con todos los operadores y administradores, la contrasena actual enmascarada, un campo para escribir la nueva contrasena, y botones para guardar o blanquear (restaurar al valor por defecto). Los cambios se sincronizan con GitHub y funcionan en todos los navegadores (Chrome, Firefox, Edge).
Importar XLSX
Permite importar datos desde un archivo Excel (.xlsx o .xls) para cargar informacion de dias anteriores de forma masiva en lugar de ingresarla manualmente uno por uno.
Capitulo 11
Exportacion de Datos
El boton de exportacion en el header permite descargar los datos del dia actual en diferentes formatos. La opcion CSV genera un archivo de valores separados por comas que se puede abrir con Excel, Google Sheets o cualquier hoja de calculo. Incluye todos los pollings, backups, procesos, novedades y pases de turno del dia seleccionado.
Desde el Dashboard (solo administradores), tambien se pueden descargar reportes en formato HTML: un reporte general del periodo seleccionado, un listado historico de pases de turno, y un reporte detallado por dia especifico. Estos reportes incluyen toda la informacion del dia con formato profesional listo para imprimir o compartir.
Capitulo 12
Notificaciones
Cuando un operador realiza el pase de turno, se genera una notificacion que aparece en la campana del header con un numero rojo indicando las no leidas. Al hacer clic en la campana se despliega un panel con todas las notificaciones, donde se puede ver el detalle de cada una. El boton "Marcar leidas" limpia el contador y quita el indicador rojo. Las notificaciones tambien se generan al agregar novedades al timeline, manteniendo informados a todos los operadores de los eventos relevantes del turno.
Capitulo 13
Sincronizacion con GitHub
Los datos se guardan automaticamente en el navegador (localStorage) y simultaneamente se sincronizan con GitHub mediante la API de Contents. El indicador de sincronizacion en el header gira cuando hay cambios pendientes de subir y se detiene cuando todo esta sincronizado. Cada dia se guarda como un archivo JSON separado en la carpeta data/ del repositorio. La sincronizacion tiene un retraso de 500 milisegundos (debounce) para evitar demasiadas peticiones mientras se escribe rapidamente.
Si la sincronizacion falla, los datos se mantienen en el navegador y se reintentan al guardar nuevamente. Para solucionar problemas de sincronizacion, verificar la conexion a internet y recargar la pagina. Los datos locales se preservan incluso si se cierra el navegador. Al volver a abrir la aplicacion, se cargan los datos locales y se sincronizan con GitHub para obtener las ultimas actualizaciones de otros operadores.
Capitulo 14
Resolucion de Problemas
"No puedo editar nada, el dia esta cerrado"
Cuando un turno completa el pase de turno, el dia se bloquea para operadores. Solo un administrador puede modificar datos en un dia cerrado. Contactar a un administrador si es necesario hacer correcciones.
"Olvide hacer el pase de turno y ya sali"
Iniciar sesion nuevamente, navegar al dia correspondiente y realizar el pase de turno. El sistema lo permitira mientras otro turno no haya cerrado el dia desde otra cuenta.
"Los datos no se ven en otra computadora"
Verificar que el indicador de sincronizacion no este girando. Si esta detenido, los datos estan en GitHub. Si el problema persiste, puede haber un problema de conexion o de cache del navegador. Probar con Ctrl+F5 para forzar una recarga limpia.
"No aparece el boton de Pase de Turno"
Asegurarse de haber iniciado sesion correctamente. El boton solo aparece despues del login. Si el problema persiste, recargar la pagina. El boton se muestra tanto para operadores como para administradores.
"Quiero corregir un dato de un dia anterior"
Si el dia no esta cerrado, se puede navegar a la fecha y editarlo normalmente. Si el dia esta cerrado, es necesario iniciar sesion como administrador para poder modificarlo.
